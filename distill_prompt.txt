你的任务是对用户查询进行意图分类，并提供详细的推理过程。你将接收三个输入：
 1.分类判断准则:
```
$criteria
```
 2.用户查询内容:
```
$query
```
3.目标分类类别:
```
$label
```
## 任务说明

你需要：

1. 忽略已知的目标分类类别，独立进行推理分析
2. 使用第一人称独白形式，详细记录你的推理过程，话术要多变，倾向正式，避免风格呆板
3. 在推理过程中引用关键判断准则，使用 `<quote>...</quote>` 标签
4. 逐一分析各个可能的分类，像人类思考一样自然展开
5. 最终得出结论
6. **确保推理过程以 `<think>` 开始并以 `</think>` 结束**

## 输出要求
详细的推理过程，需要：
- 自然的探索和验证
- 引用判断准则中的相关关键条款的内容
- 分析用户查询内容与各条款的匹配程度
- 逐步排除不符合的分类
- 得出最符合的分类(唯一)
- **确保推理过程以 `<think>` 开始并以 `</think>` 结束**

## 输出格式

```
<think>推理内容在这里</think>
<answer>最终分类结果在这里</answer>
```


## 推理要求
1. **试错性**：探索和验证
2. **独立性**：完全忽略已知的 `目标分类类别`，基于 `分类判断准则` 和 `用户查询内容` 独立推理
3. **推理完整性**：简洁解释每个判断步骤，不要跳过推理过程
4. **引用性**：引用时使用更简洁的方式，如"第x条中的<quote>相关关键内容</quote>"，尽量避免引用整个条款[必须!!!]
5. **自然性**：推理过程应像人类思考一样自然流畅
6. **格式完整性**：确保推理过程以 `<think>` 开始并以 `</think>` 结束,最终分类结果以 `<answer>` 开始并以 `</answer>` 结束

请记住，推理过程应该是真实的思考过程，而不是为了得出特定结果而设计的。**在完成推理后，请检查输出格式，确保推理内容以 `<think>` 开始并以 `</think>` 结束。**
注意:引用时不要引用整个部分，而是里面的你所涉及判断依据的内容。
